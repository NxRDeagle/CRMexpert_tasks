! При каждом новом указанном текстовом файле, он будет полностью очищен.

1. Указать несуществующий файл, например 123.txt - получить ошибку
2. Нажать BACKSPACE для смены активного файла
3. Прожать ENTER (пустой ввод) - получить ошибку
4. Указать путь не к текстовому файлу (программе, ярлыку и тд) - получить ошибку
5. Ввести путь к существующему корректному текстовому файлу
6. Ввести команду /help для отображения списка доступных команд
7. Сменить один текстовый файл на другой, введя корректный путь к файлу после команды /path
8. Ввести несуществующую команду (не из списка) - получить ошибку
9. Ввести /all - получить сообщение о том, что встреч еще нет
10. Ввести /new для создания новой встречи
11. Ввод пустого значения - получить ошибку
12. Ввод некорректной даты, например 111 12 2025 - получить ошибку
13. Ввод начальной даты позднее, чем конечной - получить ошибку
14. Ввод периода, который уже наступил относительно текущего момента времени, 
например начальная дата: "12 02 2024 13:00", конечная дата: "12 02 2024 13:20" - получить ошибку
15. Ввод корректного периода, например начальная дата: "30 08 2024" конечная дата: "30 08 2024 2:00:30"
16. Ввод пустого значения в названия встречи - ошибка
17. Ввод любого названия для встречи, кроме пробелов/пустой строки
18. Ввод времени для оповещения до встречи (пустое значение/не в формате HH:MM:SS или HH:MM - ошибка)
19. Ввод времени для оповещения до встречи, такого, что если вычесть это время из начала времени встречи, 
то этот момент уже наступил - получить ошибку
Пример: сейчас 16:00, встреча в 17:00, время до оповещения: 02:00, вычитаем - получается оповещение должно прийти в 15:00,
 а такого быть не может
! При большом значении времени генерируется исключение, связанное с переполнением, пока не исправил. использование TimeSpan
для хранения миллисекунд - результат тот же. Для Корректного выполнения пока надо брать дату, более близкую к текущей,
и не большое время до уведомления.
20. Ввести корректные даты встречи, название встречи, время до оповещения подождать, убедиться что пришло оповещение.

Например из моего теста: 
/new
26 08 2024 3:36
26 08 2024 3:37
e
00:01

Результат:

Создана новая встреча: ID = 1 | Название встречи: e | Начало встречи: 26.08.2024 3:36:00 | Окончание встречи: 26.08.2024 3:37:00 | Время оповещения о встрече: 26.08.2024 3:35:00

Нажмите ENTER чтобы продолжить, BACKSPACE для смены активного файла, или ESCAPE для завершения работы программы.

========== Уведомление: У вас запланирована встреча! ==========
ID = 1 | Название встречи: e | Начало встречи: 26.08.2024 3:36:00 | Окончание встречи: 26.08.2024 3:37:00 | Время оповещения о встрече: 26.08.2024 3:35:00

21. Ввести другую встречу, которая пересекается с уже существующей - получить ошибку (со списком пересекающихся встреч)
22. Посоздавать встречи, после чего проверить команды отображения:
/all - показать все существующие встречи, 
/com - показать  все предстоящие встречи,
/allbyname - показать все встречи, отсортированные по имени
/allbydate - показать все встречи, отсортированные по дате начала
23. Ввести команду /day, ввести день на который нет встреч - получить соответствующее сообщение
24. Ввести команду /day, пустой ввод или некорректный формат - получить ошибку
25. Ввести команду /day, дату со встречами - получить список на эту дату
26. Ввести команду /del, попробовать некорректный ввод или несуществующий id встречи - получить ошибку
27. Удалить через /del существующую встречу, увидеть сообщение о успешном удалении
28. Создать встречи, попытаться обновить у них наименования, даты, время оповещения через /upd, 
в том числе затестить некорректный ввод названий свойств, ввести 'cancel' для отмены действия.
! При обновлении встреч срабатывает уведомление, не успел это пофиксить.
29. Ввести /exp для экспорта в текстовый файл данных за определенную дату
30. Нажать ESCAPE для завершения работы программы
